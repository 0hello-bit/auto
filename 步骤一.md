  二、关键实现说明（逐文件要点）

  代码全部在磁盘上，下面是每个文件的设计要点（避免在此粘贴上千行；如需把某个文件完整贴出请告诉我）：

  - account_parser.py — 按要求用 raw.split("----", 3)，maxsplit=3 保证 refresh_token 中即使含 ----
    也完整保留（测试 test_refresh_token_with_special_chars_is_not_truncated 专门验证）。注释也标注了 token
    不做内部 strip。
  - microsoft_oauth.py — POST 到 TOKEN_URL，grant_type=refresh_token、带 scope。access_token 按 expires_in
    缓存（提前 60s 刷新），避免每 20s 都打 token 端点。日志只用 _mask() 打印 token 头尾，绝不打印完整
    token/password；若返回新的 refresh_token 会自动落库（轮换）。
  - imap_client.py — IMAP4_SSL(outlook.office365.com:993)，XOAUTH2 串 user=...\x01auth=Bearer
    ...\x01\x01；select("INBOX", readonly=True) + BODY.PEEK[]，不修改邮箱已读状态；取末尾 N 封（最旧在前，保证 DB
    id 递增 = 越新）。
  - database.py — 在文档给的 messages、poll_status 两表基础上增加了一张 accounts
    表（导入/列表/持久化所必需）。每次操作开新连接 + WAL + busy_timeout，天然支持「后台线程 + API 线程池」并发。
  - poller.py — 后台 daemon 线程每 POLL_INTERVAL_SECONDS 轮询；每邮箱一把锁避免后台与 /api/code
    同时登录同一邮箱；token 失效时自动 force-refresh 重试一次；find_code() 用请求传入的 pattern 重新从
    subject/body/html 提取并支持 subject_keyword/from_keyword 过滤。
  - api.py — APIRouter(prefix="/api", dependencies=[Depends(require_api_key)]) 统一鉴权，hmac.compare_digest
    常量时间比较；/api/code 为同步函数（FastAPI 自动放线程池），先查库→未命中则轮询→每 5s 重试→超时返回 408。
  - main.py — lifespan 里初始化 DB、加载 accounts.txt、启动/停止后台轮询；统一把 HTTPException/校验错误转成
    {"code","msg","data"} 包络；/health 免鉴权。

  四、导入邮箱示例

  curl -X POST http://127.0.0.1:5050/api/accounts/import `
    -H "x-api-key: <你的API_KEY>" `
    -H "Content-Type: application/json" `
    -d '{"text":"a@hotmail.com----pass----client_id----refresh_token\nb@hotmail.com----pass----client_id----refr
  esh_token"}'
  # -> {"code":1,"msg":"success","data":{"imported":2,"errors":[]}}

  或把账号写进 accounts.txt（每行 email----password----client_id----refresh_token）后重启自动加载。

  五、获取验证码示例

  curl -X POST http://127.0.0.1:5050/api/code `
    -H "x-api-key: <你的API_KEY>" `
    -H "Content-Type: application/json" `
    -d
  '{"email":"user@hotmail.com","timeout":180,"pattern":"\\b\\d{6}\\b","subject_keyword":"","from_keyword":""}'
  # 成功 -> {"code":1,"msg":"success","data":{"email":"user@hotmail.com","verification_code":"123456"}}
  # 超时 -> HTTP 408 {"code":408,"msg":"timeout","data":{"email":"user@hotmail.com"}}

  ▎ 建议：先触发对方发送验证码，再调用 /api/code，避免命中历史旧验证码（/api/code 返回的是最新一封匹配邮件）。

  六、常见错误排查

  ┌──────────────────────────────┬──────────────────────────────────────────────────────────────────────────┐
  │             现象             │                                   处理                                   │
  ├──────────────────────────────┼──────────────────────────────────────────────────────────────────────────┤
  │ /api/* 返回 401 unauthorized │ x-api-key 不对，或 .env 未设                                             │
  │                              │ API_KEY（未设时所有接口一律拒绝，启动日志会警告）                        │
  ├──────────────────────────────┼──────────────────────────────────────────────────────────────────────────┤
  │ last_error 含 invalid_grant  │ refresh_token 失效/过期或与 client_id 不匹配，需重新获取                 │
  ├──────────────────────────────┼──────────────────────────────────────────────────────────────────────────┤
  │ XOAUTH2 auth failed /        │ scope 缺 IMAP 权限、账号未开 IMAP，或 token 无效；确认 IMAP_SCOPE        │
  │ AUTHENTICATE failed          │                                                                          │
  ├──────────────────────────────┼──────────────────────────────────────────────────────────────────────────┤
  │ IMAP connect failed          │ 网络/防火墙，确认可达 outlook.office365.com:993                          │
  ├──────────────────────────────┼──────────────────────────────────────────────────────────────────────────┤
  │ /api/code 一直 408           │ 邮件未到 / pattern 不匹配 / 被关键词过滤；先用 GET                       │
  │                              │ /api/messages?email_addr=... 看实际邮件                                  │
  ├──────────────────────────────┼──────────────────────────────────────────────────────────────────────────┤
  │ imported 为 0                │ 账号行不是 4 段或没用 ----；看返回 errors                                │
  └──────────────────────────────┴──────────────────────────────────────────────────────────────────────────┘
