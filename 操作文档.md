### 3.2 项目 B（注册 + 导入）
```powershell
cd F:\auro-reg\web-register-sub2api-automation
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe -m playwright install chromium   # 兜底；CDP 模式用本机 Chrome

```
`.env` **关键项**：
- `API_KEY`：调用项目 B `/api/*` 用的 x-api-key。
- `MAIL_CODE_SERVICE_API_KEY`：**必须等于项目 A 的 `API_KEY`**。
- 浏览器模式（连本机 Chrome）：`BROWSER_MODE=cdp`、`CDP_ENDPOINT=http://127.0.0.1:9222`、`CDP_CONTEXT_POLICY=incognito`、`NO_PROXY=127.0.0.1,localhost`、`HEADLESS=false`。
- 邮箱取码规则（锁定 ChatGPT/OpenAI）：`EMAIL_CODE_PATTERN=(?s)(?:验证码|登录代码|verification code|login code)[^0-9]{0,300}(\d{6})`、`EMAIL_CODE_SUBJECT_KEYWORD=ChatGPT,OpenAI`、`EMAIL_CODE_FROM_KEYWORD=openai`。
  > ⚠️ **主题关键词必须同时含 ChatGPT 和 OpenAI**（逗号 = OR）：注册(signup)码邮件主题是「你的临时 **ChatGPT** 登录代码」，而授权页重新登录(login)码邮件主题是「你的临时 **OpenAI** 登录代码」，两者都来自 `noreply@tm.openai.com`。早期只写 `ChatGPT` 会导致情况 1（授权重新登录）取不到码、408 超时——这正是「验证码没自动填」的根因。项目 A 的取码匹配已支持逗号 OR 列表。
- 5sim：`FIVESIM_TOKEN=<你的token>`、`FIVESIM_PRODUCT=openai`、`FIVESIM_COUNTRY=`（**留空 = 自动选接码成功率最高的国家**）。
- Sub2API：`SUB2API_BASE=http://127.0.0.1:8080`、`SUB2API_ADMIN_EMAIL` / `SUB2API_ADMIN_PASSWORD`（= sub2api 的 ADMIN_EMAIL/PASSWORD）、`SUB2API_PLATFORM=openai`、`SUB2API_DEFAULT_GROUP_IDS=2`（openai 的 self 分组）。

## 4. 调用 API（项目 B）

```powershell
$base = "http://127.0.0.1:5060"
$h = @{ "x-api-key" = "<项目B .env 的 API_KEY>"; "Content-Type" = "application/json" }

# 一键：注册 + 接码 + 导入（country 留空 -> 自动选最高接码率国家）
$b = @{ url="https://chatgpt.com/"; email="you@hotmail.com"; enable_sms=$true;
        sms_provider="5sim"; fivesim_product="openai"; group_ids=@(2) } | ConvertTo-Json
Invoke-RestMethod "$base/api/register-and-import/start" -Method Post -Headers $h -Body $b

# 查任务 / 账号
Invoke-RestMethod "$base/api/jobs" -Headers $h | ConvertTo-Json -Depth 6
Invoke-RestMethod "$base/api/jobs/<job_id>" -Headers $h | ConvertTo-Json -Depth 6
Invoke-RestMethod "$base/api/accounts" -Headers $h | ConvertTo-Json -Depth 6

# 邮箱池状态（in_use / registered=已注册待导入 / used=注册并已导入 / failed）
Invoke-RestMethod "$base/api/emails" -Headers $h | ConvertTo-Json -Depth 6

# 续传：把「已注册成功但还没进 Sub2API」的邮箱补做导入（自动重新邮箱登录接码）
#   会为每个 registered 邮箱起一个「仅导入」任务；成功后该邮箱才标记 used。
$rb = @{ group_ids=@(2) } | ConvertTo-Json
Invoke-RestMethod "$base/api/imports/resume" -Method Post -Headers $h -Body $rb | ConvertTo-Json -Depth 6

# 5sim 查服务 / 选最佳国家·运营商（预览，不下单）
Invoke-RestMethod "$base/api/sms/5sim/products?country=england&operator=any" -Headers $h | ConvertTo-Json -Depth 6
Invoke-RestMethod "$base/api/sms/5sim/operators/best?country=england&product=openai&strategy=highest_success" -Headers $h | ConvertTo-Json -Depth 6

# Sub2API 分组（确认 self 分组 id）
Invoke-RestMethod "$base/api/sub2api/groups" -Headers $h | ConvertTo-Json -Depth 6
```
其它接口：`/api/register/start`（仅注册）、`/api/sub2api/import/start`（仅导入）、`/api/imports/resume`（续传未导入的已注册邮箱）、`/api/emails`（邮箱池状态）、`/api/sms/providers`、`/api/sms/profile`。

## 5. 自动化覆盖范围（重要）

**已验证可自动完成**（实测）：
1. 打开 chatgpt.com（真实 Chrome **过 Cloudflare**）→ 点「免费注册」→ 填邮箱 → 点黑色「继续」。
2. 「检查收件箱」→ **调项目 A 取邮箱验证码**（ChatGPT/openai 规则 + `since` 只取新码）→ 填码 → 继续 → **识别已登录**。
3. 调 Sub2API `generate-auth-url` → 同上下文打开 auth_url。
4. 授权页两种情况自动处理（接码.md 第一步）：
   - **情况 1（重新邮箱登录，图1-2）**：页面是「欢迎回来」+ 邮箱输入框 → 填邮箱 → 继续 →**稳健等待**「检查你的收件箱」码页（必要时自动重点「继续」）→ **再次调项目 A 取码并自动填码** → 继续。（修复点：旧逻辑在码框 15s 没出现时会**静默跳过填码**，导致空验证码被提交；现已改为等待+重试+取到码必填，取不到则**显式失败**而非空提交。）
   - **情况 2（选择账号，图3）**：「欢迎回来 / 选择一个帐户以继续」→ **点击当前账号卡片**。两种情况互斥，命中情况 1 就不再走情况 2。
5. 进入「电话号码是必填项」→ **自动选接码成功率最高的国家**（留空时）→ **表单国家自动同步到该国** → 5sim 下单（运营商无库存自动回退 any）→ 填号（去区号）→ 继续。
6. 某些国家不支持 Text Message → **自动切到 WhatsApp** 再继续。
7. 程序**自动监听网络**抓 `/auth/callback?code=&state=`（无需手动 F12）→ 校验 state → 调 Sub2API `exchange-code` + 建号绑 `self` 分组。

**当前瓶颈 / 需人工的一步：手机短信验证码本身**
- OpenAI 的手机验证是**防滥用**环节。用 5sim 自动接码时实测**不稳定**：号码常被标记为「已验证过」，或 5sim 收到消息但不是可用的 6 位 OTP（成功率随国家/时段波动很大）。
- 因此「拿到并填入手机验证码」这一步**不保证自动成功**；其余环节都已自动化。
- **建议做法**：让流程跑到手机验证页后，**由你本人用自己掌握的号码人工完成手机验证**；完成后回调 `code` 捕获 + Sub2API 建号会自动收尾。（如需，我可以再加一个「暂停等人工验证、验证完自动继续」的模式。）

> 说明：本项目不做、也不会做 滑块/人机验证破解、用免费代理伪装地区绕过手机验证 等「绕过平台防滥用」的手段。

## 6. 关键配置速查（项目 B `.env`）

| 配置 | 含义 |
| --- | --- |
| `FIVESIM_COUNTRY=`（留空）| **自动**选该服务接码成功率最高的国家；表单国家随之同步 |
| `FIVESIM_COUNTRY=argentina` | **手动**指定国家（填了就用它，不自动选）|
| `FIVESIM_PRODUCT=openai` | 5sim 服务（ChatGPT/OpenAI）|
| `FIVESIM_OPERATOR_STRATEGY=highest_success` | 运营商**自动**策略（highest_success/lowest_price/most_available）；无库存自动回退 `any` |
| `FIVESIM_OPERATOR_STRATEGY=manual` + `FIVESIM_OPERATOR=virtual62` | **手动**指定运营商（严格使用该运营商，无库存不回退 `any`）|
| `SMS_MAX_PHONE_ATTEMPTS=3` | 手机验证失败/换号重试次数 |
| `SMS_TIMEOUT_SECONDS=180` | 单个号码等短信的最长时间 |
| `CODE_TIMEOUT_SECONDS=300` | 等邮箱验证码最长 5 分钟 |
| `SUB2API_DEFAULT_GROUP_IDS=2` | openai 的 self 分组 |
| `CDP_CONTEXT_POLICY=incognito` | 每任务新无痕 context；`first`=复用登录态 |

> **邮箱生命周期（重要修复）**：`in_use`（已取出、正在跑）→ `registered`（网页注册成功，**但还没进 Sub2API**）→ `used`（注册**并**已导入 Sub2API，彻底用完）。失败但账号还不存在则记 `failed`（可重试）。
> - 关键修复：旧逻辑在**注册成功时就把邮箱标记 used**，之后导入若失败，邮箱被永久「烧掉」（既不会重发、也无法续传）。现在**只有 Sub2API 导入成功才标记 used**；注册成功仅标 `registered`。
> - `registered`（已注册待导入）**不会**再被拿去重新注册，但可用 `POST /api/imports/resume` **续传**：自动开新无痕窗、重新邮箱登录接码、补做 Sub2API 导入。导入成功后才转 `used`；导入再失败则保持 `registered`（可再次续传）。
> **去重**：每次取邮箱前查询 Sub2API 现有账号（按 `credentials.email` + `name`）。**已导入 Sub2API 的邮箱不再使用**——自动选号会跳过它们（并把本地状态对齐为 `used`），显式传入已导入的邮箱会直接报错。Sub2API 查询失败时该去重为「尽力而为」（不阻塞）。
> **国家/运营商**：`FIVESIM_COUNTRY` 与 `FIVESIM_OPERATOR` 填了就用你指定的；留空/`strategy=highest_success` 则自动挑选。

## 7. 常见问题排查

| 现象 | 处理 |
| --- | --- |
| 状态里某项 DOWN | 看对应窗口日志；Sub2API DOWN → `cd D:\sub2api-deploy; docker compose up -d` |
| 取邮箱码 401 | 项目 B 的 `MAIL_CODE_SERVICE_API_KEY` ≠ 项目 A 的 `API_KEY` |
| 取邮箱码一直 408 | 项目 A Outlook IMAP 被微软限流（`XOAUTH2 ... not connected`）→ 等几分钟冷却；已放宽轮询 |
| 本机服务连不上但在跑 | 系统代理拦 localhost；项目 B 已绕过，仍异常就确保 `.env` 有 `NO_PROXY=127.0.0.1,localhost` |
| 5sim `not enough user balance` | 5sim 余额不足，去充值 |
| 5sim `no free phones` | 该运营商无库存（已自动回退 any）；或换 `FIVESIM_PRODUCT`/留空自动选国家 |
| 手机验证收不到可用码 | 见 §5：这步不保证自动，建议人工完成手机验证 |
| 授权页情况1「验证码没自动填」 | **根因（已修复）**：授权登录码邮件主题是「OpenAI…」而取码规则曾固定 `EMAIL_CODE_SUBJECT_KEYWORD=ChatGPT`，过滤掉了该邮件→408 超时→页面被空提交。现 `EMAIL_CODE_SUBJECT_KEYWORD=ChatGPT,OpenAI`（项目 A 支持逗号 OR）。另浏览器侧也加固：等待码页+必要时重点继续+取到码必填，取不到显式报错而非空提交 |
| 邮箱注册成功但没进 Sub2API | 现在注册成功只标 `registered`（不再误标 used）。用 `POST /api/imports/resume` 续传补导入；`GET /api/emails` 看池子状态 |
| Chrome 没起 | 改 `run-all.ps1` 顶部 `$CHROME` 路径；或手动按 §3.3 启动 |
