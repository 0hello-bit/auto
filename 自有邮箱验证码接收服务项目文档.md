# 自有邮箱验证码接收服务项目文档

## 1. 项目名称

Owned Mail Verification Code Service

中文名：自有邮箱验证码接收服务

## 2. 项目目标

本项目用于批量接入用户自己拥有或已授权管理的邮箱账号，实时轮询邮件，自动提取验证码，并通过 HTTP API 提供给外部系统调用。

项目只用于自有业务系统、测试系统、内部自动化流程中的邮件验证码读取，不用于第三方平台的批量注册、绕过验证或规避风控。

## 3. 核心功能

### 3.1 邮箱账号批量导入

支持以下格式批量导入邮箱账号：

```text
email----password----client_id----refresh_token
```

字段说明：

```text
email：邮箱地址，例如 user@hotmail.com
password：邮箱密码，主要用于记录或兼容，不建议直接用于登录
client_id：Microsoft OAuth 应用 client_id
refresh_token：用于刷新 access_token 的 Microsoft OAuth refresh_token
```

支持通过以下方式导入：

```text
1. 本地 accounts.txt 文件
2. Web API 导入
3. 后续可扩展 Web 管理后台导入
```

### 3.2 邮箱轮询

系统后台定时轮询所有已导入邮箱，拉取最新邮件，并保存到本地数据库。

默认轮询周期：

```text
20 秒
```

可通过配置文件修改。

轮询逻辑：

```text
1. 读取邮箱账号列表
2. 使用 refresh_token 换取 access_token
3. 使用 IMAP XOAUTH2 登录邮箱
4. 拉取 INBOX 最新邮件
5. 解析邮件主题、发件人、正文、HTML 正文
6. 自动提取验证码
7. 写入 SQLite 数据库
8. 记录轮询状态和错误信息
```

### 3.3 验证码提取

默认验证码正则：

```regex
\b\d{4,8}\b
```

支持调用 API 时自定义正则，例如：

```regex
\b\d{6}\b
```

验证码提取范围：

```text
1. 邮件主题 subject
2. 邮件正文 body
3. HTML 正文 html_body
```

支持按条件过滤：

```text
subject_keyword：邮件标题关键词
from_keyword：发件人关键词
pattern：验证码匹配正则
timeout：最长等待时间
```

### 3.4 外部 API 调用

外部系统可以调用 API 获取指定邮箱收到的验证码。

典型调用流程：

```text
1. 外部系统触发发送邮件验证码
2. 外部系统调用本服务 /api/code
3. 本服务等待并轮询目标邮箱
4. 找到验证码后返回 verification_code
5. 外部系统继续执行后续验证流程
```

### 3.5 邮件缓存与查询

系统会把拉取到的邮件保存到 SQLite 数据库，避免重复登录邮箱和重复解析邮件。

保存字段：

```text
email
message_id
from_addr
subject
body
html_body
date_raw
code
created_at
```

### 3.6 轮询状态查看

系统提供 API 查看每个邮箱的状态：

```text
邮箱地址
是否有 refresh_token
最近轮询时间
最近错误信息
```

## 4. 技术栈

### 4.1 后端框架

```text
Python 3.10+
FastAPI
Uvicorn
```

### 4.2 邮箱协议

```text
IMAP over SSL
Microsoft OAuth2
XOAUTH2
```

默认 IMAP 服务：

```text
outlook.office365.com:993
```

```text
https://login.microsoftonline.com/consumers/oauth2/v2.0/token
```

默认 IMAP Scope：

```text
https://outlook.office.com/IMAP.AccessAsUser.All offline_access
```

```text
SQLite
```

数据库文件：

```text
mail_codes.db
```

建议使用 `.env` 或 `config.yaml`。

核心配置项：

```text
API_KEY=强随机接口密钥

POLL_INTERVAL_SECONDS=20
LATEST_MAIL_LIMIT=20
```

## 6. API 设计

### 6.1 健康检查

```http
GET /health
```

响应：

```json
{
  "code": 1,
  "msg": "ok"
}
```

### 6.2 批量导入邮箱

```http
POST /api/accounts/import
Header: x-api-key: <API_KEY>
Content-Type: application/json
```

请求：

```json
{
  "text": "a@hotmail.com----pass----client_id----refresh_token\nb@hotmail.com----pass----client_id----refresh_token"
}
```

响应：

```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "imported": 2
  }
}
```

要求：

```text
1. 每行一个邮箱账号
2. 使用 ---- 分隔
3. refresh_token 可能包含特殊字符，解析时必须只 split 前 3 个分隔符
4. 导入前校验格式
5. 不在响应里返回 password 或 refresh_token
```

### 6.3 查看邮箱列表

```http
GET /api/accounts
Header: x-api-key: <API_KEY>
```

响应：

```json
{
  "code": 1,
  "msg": "success",
  "data": [
    {
      "email": "user@hotmail.com",
      "client_id": "xxx",
      "has_refresh_token": true,
      "last_poll_at": 1780000000,
      "last_error": ""
    }
  ]
}
```

### 6.4 手动触发轮询

```http
POST /api/poll
Header: x-api-key: <API_KEY>
```

响应：

```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "ok": 1,
    "failed": []
  }
}
```

### 6.5 获取验证码

```http
POST /api/code
Header: x-api-key: <API_KEY>
Content-Type: application/json
```

请求：

```json
{
  "email": "user@hotmail.com",
  "timeout": 180,
  "pattern": "\\b\\d{6}\\b",
  "subject_keyword": "",
  "from_keyword": ""
}
```

响应成功：

```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "email": "user@hotmail.com",
    "verification_code": "123456"
  }
}
```

响应超时：

```json
{
  "code": 408,
  "msg": "timeout",
  "data": {
    "email": "user@hotmail.com"
  }
}
```

### 6.6 查询最近邮件摘要

```http
GET /api/messages?email_addr=user@hotmail.com
Header: x-api-key: <API_KEY>
```

响应：

```json
{
  "code": 1,
  "msg": "success",
  "data": [
    {
      "email": "user@hotmail.com",
      "from_addr": "noreply@example.com",
      "subject": "Your verification code",
      "code": "123456",
      "date_raw": "...",
      "created_at": 1780000000
    }
  ]
}
```

要求：

```text
1. 默认不返回完整邮件正文
2. 只返回摘要和提取到的 code
3. 后续可添加 admin 参数查看完整邮件，但默认关闭
```

## 7. 数据库设计

### 7.1 messages 表

```sql
CREATE TABLE IF NOT EXISTS messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    email TEXT NOT NULL,
    message_id TEXT,
    from_addr TEXT,
    subject TEXT,
    body TEXT,
    html_body TEXT,
    date_raw TEXT,
    code TEXT,
    created_at INTEGER NOT NULL,
    UNIQUE(email, message_id)
);
```

### 7.2 poll_status 表

```sql
CREATE TABLE IF NOT EXISTS poll_status (
    email TEXT PRIMARY KEY,
    last_poll_at INTEGER,
    last_error TEXT
);
```

## 8. 安全要求

### 8.1 接口鉴权

所有 `/api/*` 接口必须要求：

```http
x-api-key: <API_KEY>
```

未授权返回：

```json
{
  "code": 401,
  "msg": "unauthorized"
}
```

### 8.2 敏感信息保护

以下内容不能出现在 API 响应中：

```text
邮箱密码
refresh_token
access_token
完整 Authorization header
完整邮件正文，除非明确开启 debug/admin 模式
```

### 8.3 文件安全

以下文件必须加入 `.gitignore`：

```text
accounts.txt
.env
mail_codes.db
*.log
```

### 8.4 默认监听地址

默认只监听：

```text
127.0.0.1
```

不要默认暴露到公网。

如需公网访问，必须额外配置：

```text
HTTPS
强 API Key
IP 白名单
反向代理访问控制
```

## 9. 使用方式

### 9.1 安装依赖

```bash
pip install -r requirements.txt
```

### 9.2 启动服务

```bash
uvicorn app.main:app --host 127.0.0.1 --port 5050
```

### 9.3 导入邮箱

```bash
curl -X POST http://127.0.0.1:5050/api/accounts/import \
  -H "x-api-key: your-secret" \
  -H "Content-Type: application/json" \
  -d '{"text":"user@hotmail.com----password----client_id----refresh_token"}'
```

### 9.4 获取验证码

```bash
curl -X POST http://127.0.0.1:5050/api/code \
  -H "x-api-key: your-secret" \
  -H "Content-Type: application/json" \
  -d '{"email":"user@hotmail.com","timeout":180,"pattern":"\\b\\d{6}\\b"}'
```

