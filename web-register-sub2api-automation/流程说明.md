# 从头到尾的逻辑与流程（注册 + Sub2API 导入）

> 供你审阅。已按 `接码.md`（含 6 张图 / 两种情况）对齐代码。下面分「组件 → 一键流程 → 关键选择器 → 已验证/待验证 → 已知风险」。

## 0. 组件与端口
| 组件 | 作用 | 地址 |
| --- | --- | --- |
| 项目 A（owned-mail-code-service）| 收 ChatGPT/OpenAI 邮件、按规则取验证码 | 127.0.0.1:5050 |
| 项目 B（本服务）| 编排全流程、驱动浏览器 | 127.0.0.1:5060 |
| 本机 Chrome（CDP）| 真实浏览器（过 Cloudflare）| 127.0.0.1:9222 |
| 5sim | 接码（买号、收短信）| 5sim.net（走系统代理）|
| Sub2API | 管理 API、建号 | 127.0.0.1:8080 |

- 浏览器：`BROWSER_MODE=cdp` + `CDP_CONTEXT_POLICY=incognito`。每个任务新建**无痕 context**，任务结束关闭并清空 Cookie/localStorage。**注册页与授权页用同一个 context**（保留登录态）。
- 邮箱取码固定规则：`pattern=(?s)(?:验证码|登录代码|verification code|login code)[^0-9]{0,300}(\d{6})`、`subject=ChatGPT,OpenAI`、`from=openai`，并带 `since`（只取「点继续之后」新发的码，避免旧码）。
  - ⚠️ **主题关键词必须是 `ChatGPT,OpenAI`（逗号=OR）**：signup 码邮件主题含「ChatGPT」，而授权页 login 码邮件主题含「OpenAI」（都来自 `noreply@tm.openai.com`）。只写 `ChatGPT` 会过滤掉授权登录码→项目A 408→「验证码没自动填」。项目 A `poller.find_code` 已支持逗号 OR 列表匹配 subject/from。

## 1. 一键入口：`POST /api/register-and-import/start`
请求体：`url、email、enable_sms、sms_provider=5sim、fivesim_country=argentina、fivesim_product=openai、group_ids=[2]`。
分两大阶段，**同一无痕 context**。

### 阶段一　网页注册（register_worker）
1. 打开 `REGISTER_URL`（chatgpt.com），等待页面 hydrate（networkidle/短延时，避免点击无效）。
2. 点击「**免费注册**」（`button:has-text("免费注册")`）。
3. 弹出「登录或注册」→ 填邮箱（`input#email`）→ 点**黑色「继续」**（`button[type="submit"]:has-text("继续")`，**排除**"使用 Google/Apple 账户继续"）。
4. 进入「检查你的收件箱」→ 记下时间戳 `since` →**调项目 A 取邮箱验证码**（ChatGPT/openai 规则，最长 5 分钟）→ 填验证码（`input[name="code"]`）→ 点「继续」。
5. 之后：**若出现姓名/年龄页**则填（可选）；**否则识别为「已登录」**直接判注册成功（当前 ChatGPT 流程是验证码后直接登录、无姓名/年龄页）。
6. 注册成功 → 同一 context 已登录 ChatGPT。

### 阶段二　Sub2API 导入（sub2api_import_worker）
1. 调 **Sub2API `generate-auth-url`** → 得 `auth_url`（`https://auth.openai.com/oauth/authorize?...&redirect_uri=http://localhost:1455/auth/callback&state=...`）+ `session_id`；从 auth_url 解析 `expected_state`。
2. 同一 context **新开 page_auth**，挂**网络监听**（抓任何含 `/auth/callback?code=` 的 URL）→ 打开 auth_url。
3. **打开后两种情况（接码.md）——程序自动判断**：
   - **情况 1（图1-2，重新登录）**：页面是「欢迎回来」+ 邮箱输入框 → 填邮箱 → 点「继续」→**稳健等待**「检查你的收件箱」码页（最长 ~30s；若邮箱框还在/码框没出现，自动**重点一次「继续」**）→**再次调项目 A 取邮箱码**→ 填「验证码」→ 点「继续」→ 校验是否已离开码页。
     （判定：页面存在邮箱输入框 → 走情况 1；命中情况 1 后**不再走情况 2**）
     （**已修复的 bug**：① **主因**——授权登录码邮件主题是「OpenAI」，而取码规则曾固定 `subject=ChatGPT`，把该邮件过滤掉→项目A 408 超时（实测：项目A 已收到邮件，但 B 取不到码）。已改为 `subject=ChatGPT,OpenAI`。② **加固**——旧浏览器逻辑用 15s `resolve_locator` 预检码框，超时即 `return` **静默跳过取码/填码**，随后被「best-effort 继续」点成**空验证码提交**。现改为：等待码页+必要时重点继续+**取到码必填**；码页始终不出现则**显式抛错**让导入失败，邮箱保持 `registered` 可续传。）
   - **情况 2（图3，选择账号）**：「欢迎回来 / 选择一个帐户以继续」→ **点击当前账号卡片**（`button:has-text("<邮箱@前缀>")`，已实测唯一命中）。
     （判定：无邮箱输入框、有账号卡片 → 走情况 2；有时 OpenAI 会自动跳过此页）
4. 若出现 consent/「继续」页 → 点「继续/授权」（`button[type="submit"]:has-text("继续")` 等）。
5. **若出现「电话号码是必填项」→ 手机验证（带重试，最多 `SMS_MAX_PHONE_ATTEMPTS=3` 次）**：
   1. **确定接码国家**：`FIVESIM_COUNTRY` 有值就用它；**留空则自动选「该服务接码成功率最高」的国家**（调 5sim `guest/prices?product=`，实测 openai → england 47.55%，远高于 argentina 4.76%）。
   2. **国家下拉**：把 OAuth 手机表单的国家**自动切到与购买国家一致**——当前是**原生 `<select>`**，按 中文名/英文名/区号(+44) 匹配选项选中（已实测原生 select 选中成功）。
   3. **5sim 下单号码**：按策略选运营商（`highest_success`），**若该运营商无库存（no free phones）→ 回退 `operator=any`**（已实测成功）。
   4. 填手机号（**去掉区号前缀**，因表单已选国家会自带 +XX）→ 点「继续」。
   5. 若立刻提示「**该号码已被验证过**」等（`PHONE_ERROR_KEYWORDS`）→ **取消该订单、返回上一页、换新号重试**。
   6. 否则进入「查看你的手机」→ **轮询 5sim 取短信验证码**→ 填「代码」（`input[name="code"]`）→ 点「继续」。
   7. 验证**失败**（仍停在代码页 / 出现错误字样）→ **取消、返回、换号重试**；**成功**→ 完成（finish）订单。
6. 进入「**使用 ChatGPT 登录到 Codex**」consent → 点「继续」。
7. **程序自动从网络捕获** `/auth/callback?code=...&state=...`（**无需手动 F12**；即你文档里 F12 看到的那条 callback）。
8. **校验 `state == expected_state`**，不一致立即失败。
9. 调 **Sub2API**：`exchange-code(session_id, code, state)` → 拿到 OAuth 凭据 → `POST /admin/accounts`（`platform=openai, type=oauth, credentials, group_ids=[2]=self, concurrency, priority`）→ **建号成功**。
10. 成功后 finish 短信订单；失败/超时 cancel 订单；结果落库；失败存截图。
11. **邮箱生命周期**：注册成功仅标 `registered`（**不再**在注册成功时就标 `used`）；**只有 Sub2API 导入成功**才把邮箱标 `used`。导入失败时邮箱保持 `registered`，可用 `POST /api/imports/resume` 续传（自动重新邮箱登录接码→补做导入）。`GET /api/emails` 查看池子状态。

## 2. 关键选择器 / 配置（.env，均可改）
- 免费注册：`button:has-text("免费注册")`
- 邮箱框：`input#email, input[name="email"], input[placeholder="电子邮件地址"]`
- 邮箱「继续」/邮箱码「继续」：`button[type="submit"]:has-text("继续")`（排除 SSO）
- 邮箱码框：`input[name="code"], input[autocomplete="one-time-code"]`
- 账号卡片：`button:has-text("<邮箱@前缀>")`（运行时按邮箱拼）
- 手机国家下拉：原生 `select`（`PHONE_COUNTRY_SELECT_SELECTOR=select`）；映射 argentina→阿根廷/+54
- 手机号框：`input[type="tel"]`，填入时去掉 `+54`
- 短信码框：`input[name="code"], input[placeholder="代码"/"验证码"]`
- consent 继续：`button[type="submit"]:has-text("继续"), …授权/Authorize/Allow`
- 重试：`SMS_MAX_PHONE_ATTEMPTS=3`、`PHONE_ERROR_KEYWORDS=已被验证,已验证过,已被使用,…`、`PHONE_BACK_SELECTOR=…返回/上一步…`

- 免费注册 → 邮箱 → 正确黑色「继续」（不再误点 Google）→ 邮箱码（项目 A 真实取到）→ 识别「已登录」 ✅

- 账号卡片选择 / 自动跳过 → 到达「电话号码是必填项」页 ✅

- 5sim 下单（运营商无库存自动回退 any）✅

**尚未跑通到底（待实测）⚠️：**
- 情况 1（授权页要求重新邮箱登录 + 自动填码）——**已修复静默跳过填码的 bug**（见 §1 阶段二 step3 情况1）；逻辑已就绪，建议用 `POST /api/imports/resume` 对一个 `registered` 邮箱实跑验证。

- 最终「使用 ChatGPT 登录到 Codex」consent 的「继续」点击 + 回调捕获 + `exchange-code`/建号（这几步逻辑已就绪，但还没真正走到）。

## 4. 已知风险 / 依赖
- **项目 A Outlook IMAP 偶发限流**（XOAUTH2 "User is authenticated but not connected"）——已放宽轮询（按需 15s、后台 60s），高频跑仍可能被微软限流，需冷却。
- **argentina/openai 5sim 成功率很低（1–5%）**——短信到达率低，可能要多次换号或**换国家/产品**（rate 更高的）。
- 「已被验证过」「返回上一页」「最终 consent」这三处 UI 我**还没在实跑中亲眼见到**，selector 为推测，可能需要你提供截图再微调（沿用你「卡住了问我」的方式）。
