### 1.2 代理环境特别说明（重要）

本机检测到系统代理：`HTTP_PROXY=http://127.0.0.1:7890`。这类代理（Clash/V2Ray 等）会把 **localhost** 的请求也吞掉，导致项目 B 连不上本机的项目 A（5050）和 Sub2API（8080）。

- 项目 B **已在代码层自动处理**：对 `127.0.0.1 / localhost` 目标自动绕过代理（直连），对外网（5sim / 62-US）仍走系统代理（国内访问通常需要）。无需你额外配置。
- 若仍遇到本机服务连不上，可再加保险：启动前设 `NO_PROXY`：

```powershell
$env:NO_PROXY = "127.0.0.1,localhost"
```

## 3. 配置项目 B（注册 + 接码 + 导入）

编辑 `F:\auro-reg\web-register-sub2api-automation\.env`（已生成，按需改）：

```text
# 本服务鉴权（自定义；调用 /api/* 时放到 x-api-key 头里）
API_KEY=local-register-key-please-change

# 与项目 A 对接：BASE 指向项目 A，KEY 必须等于项目 A 的 API_KEY
MAIL_CODE_SERVICE_BASE=http://127.0.0.1:5050
MAIL_CODE_SERVICE_API_KEY=change-this-to-a-strong-secret
# 邮箱验证码提取：固定用 ChatGPT/OpenAI 规则，避免取到别的邮件里的错码
EMAIL_CODE_PATTERN=(?s)(?:验证码|登录代码|verification code|login code)[^0-9]{0,300}(\d{6})
EMAIL_CODE_SUBJECT_KEYWORD=ChatGPT
EMAIL_CODE_FROM_KEYWORD=openai

# 目标注册页（你自己的项目）
REGISTER_URL=https://your-public-register-url.example.com
HEADLESS=false               # 先用 false 观察浏览器

# 短信平台：默认 5sim
ENABLE_SMS=true
SMS_PROVIDER=5sim
FIVESIM_TOKEN=你的-5sim-token        # 下单需要；查服务/查运营商用 guest 接口可不依赖它
FIVESIM_COUNTRY=argentina
FIVESIM_PRODUCT=google
FIVESIM_OPERATOR_STRATEGY=highest_success

# Sub2API（weishaw/sub2api，本地部署，JWT 管理员登录）
SUB2API_BASE=http://127.0.0.1:8080
SUB2API_ADMIN_EMAIL=admin@example.com        # = sub2api .env 的 ADMIN_EMAIL
SUB2API_ADMIN_PASSWORD=Hf133288..            # = sub2api .env 的 ADMIN_PASSWORD
SUB2API_PLATFORM=openai
SUB2API_ACCOUNT_TYPE=oauth
SUB2API_REDIRECT_URI=http://localhost:1455/auth/callback
SUB2API_DEFAULT_GROUP_IDS=2                  # openai 的 self 分组（实测 id=2）
```

> **Sub2API 对接说明（已实测）**：你的 `D:\sub2api-deploy` 是 `weishaw/sub2api`，管理 API 用 **JWT 登录**（邮箱+密码），不是 x-api-key。项目 B 已据此对接：
> 1. `POST /api/v1/auth/login` 取 token；
> 2. `GET /api/v1/admin/groups/all` → 实测 `self` 分组 `id=2, platform=openai`；
> 3. `POST /api/v1/admin/openai/generate-auth-url` → 返回 `auth.openai.com` 授权链接 + session_id；
> 4. `POST /api/v1/admin/openai/exchange-code` {session_id, code, state} → OAuth 凭据；
> 5. `POST /api/v1/admin/accounts` {platform:"openai", type:"oauth", credentials, group_ids:[2], ...} → 建号并绑到 self 分组。
>
> 你只需在项目 B `.env` 填 `SUB2API_ADMIN_EMAIL` / `SUB2API_ADMIN_PASSWORD`（与 sub2api 的 `.env` 一致）和 `SUB2API_DEFAULT_GROUP_IDS=2`。

> **联调最关键的一行**：`MAIL_CODE_SERVICE_API_KEY`（项目 B）必须 == `API_KEY`（项目 A）。否则项目 B 调项目 A 会 401。

## 5. 联通性自检（建议每次联调先跑一遍）

先准备好请求头变量（项目 B 的 key）：


```powershell

```

**5.3 项目 B → 项目 A 取邮箱验证码（核心联调）：**

```powershell
$mh = @{ "x-api-key" = "change-this-to-a-strong-secret"; "Content-Type"="application/json" }
$mb = @{ email = "PluttMinucci3076@hotmail.com"; timeout = 60; pattern = "\b\d{6}\b" } | ConvertTo-Json
# 直接验证项目 A 接口（与项目 B 内部调用同一契约）
Invoke-RestMethod "http://127.0.0.1:5050/api/code" -Method Post -Headers $mh -Body $mb
# 期望：{code=1, data={email, verification_code}}；若暂时没有新码会返回 408 timeout
```

> 实测该接口返回过真实验证码 `402100`。注册流程里，项目 B 会在填完邮箱、点"继续"后自动调用它来取码。

## 9. 常见问题快速定位

| 现象 | 处理 |
| --- | --- |
| 调项目 A 取码返回 401 | 项目 B `.env` 的 `MAIL_CODE_SERVICE_API_KEY` 与项目 A `.env` 的 `API_KEY` 不一致 |
| 取码返回 404 account not found | 该邮箱没在项目 A 的 `accounts.txt`/数据库里；先在项目 A 导入该账号 |
| 取码返回 408 timeout | 暂时没收到匹配验证码；确认项目 A 能正常收信（IMAP/OAuth 有效），或增大 `timeout` |
| 本机服务连不上但服务在跑 | 系统代理拦截 localhost；项目 B 已自动绕过，仍异常就设 `NO_PROXY=127.0.0.1,localhost` |
| `FIVESIM_PRODUCT 未配置，且请求中没有传 fivesim_product` | `.env` 设 `FIVESIM_PRODUCT` 或请求传 `fivesim_product` |
| `FIVESIM_COUNTRY 未配置，且请求中没有传 fivesim_country` | `.env` 设 `FIVESIM_COUNTRY` 或请求传 `fivesim_country` |
| 5sim `no free phones` / 余额不足 | 换 `fivesim_product` / `fivesim_country` / 策略，或充值；先用 `/operators/best` 看库存与成功率 |
| `/api/sms/profile` 报错 | 需要有效 `FIVESIM_TOKEN`（profile 是登录态接口；guest 的 products/best 不需要） |
| Sub2API 4xx/5xx 或登录失败 | 确认 Sub2API 已起；`SUB2API_ADMIN_EMAIL`/`SUB2API_ADMIN_PASSWORD` 与 sub2api `.env` 的 `ADMIN_EMAIL`/`ADMIN_PASSWORD` 一致；用 `/api/sub2api/groups` 验证 |
| 注册页元素点不到（selector not ready） | 按真实页面改 `.env` 的 `*_SELECTOR`，`HEADLESS=false` 观察 |
| 授权回调没抓到（timeout） | 增大 `timeout`、确认 `SUB2API_REDIRECT_URI` 与 Sub2API 一致；看 `traces/<job>.zip` 复盘 |

## 10. 一句话流程回顾

打开注册页 → 免费注册 → 填邮箱 → **调项目 A 取邮箱验证码** → 填码 → 填姓名年龄 → 完成注册 → 调 Sub2API 拿 auth_url/session_id → 同上下文打开 auth_url → 点继续 →（如需手机号）**按 5sim 策略自动选运营商并下单 → 填号 → 轮询取短信码 → 回填** → 监听网络抓 `/auth/callback?code=&state=` → 校验 state → 调 Sub2API `create-from-oauth` 建号绑组 → 成功 finish 订单 / 失败 cancel 订单 → 落库。
